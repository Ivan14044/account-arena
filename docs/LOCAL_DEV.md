# Как запустить сайт у себя на компьютере

Эта инструкция — для локального запуска (чтобы посмотреть/потыкать сайт). Ничего знать не нужно: одна команда.

## Что нужно один раз установить

1. **Docker Desktop** — https://www.docker.com/products/docker-desktop/
   Установи и запусти (значок кита в строке меню должен «успокоиться»).
2. Тулчейн проекта (`~/.local/toolchain`) уже стоит на этой машине. Если вдруг нет — скажи, поставлю.

> PHP, Node и Composer ставить отдельно **не надо** — они уже внутри тулчейна.

## Запуск — одна команда

Открой Терминал, перейди в папку проекта и выполни:

```bash
./scripts/start.sh
```

Первый раз это займёт несколько минут (скачиваются база данных и зависимости). В конце увидишь готовые ссылки. Дальше просто открой в браузере:

- **Сайт:** http://localhost:3000
- **Админка:** http://localhost:8000/admin
  - логин: `admin@mail.com`
  - пароль: `123123123`

## Остановить

```bash
./scripts/stop.sh          # остановить сайт (база сохраняется)
./scripts/stop.sh --all    # остановить ещё и базу данных
```

## Если что-то сломалось

- **Хочу базу с нуля (чистые данные):**
  ```bash
  ./scripts/start.sh --fresh
  ```
- **Docker не запущен** — открой Docker Desktop и подожди, пока он стартует, потом снова `./scripts/start.sh`.
- **Посмотреть, что пошло не так** — логи лежат в `.dev-state/logs/backend.log` и `.dev-state/logs/frontend.log`.

## Что под капотом (для любопытных)

- **MySQL + Redis** крутятся в Docker (файл `docker-compose.yml`). Это база данных и кэш.
- **Сайт (PHP) и фронтенд (Vite)** запускаются напрямую через тулчейн — так же, как проект проверяется тестами.
- Платежи, Google-вход и Telegram локально **не настроены** (ключи пустые) — для просмотра сайта они не нужны.
- Это **только для разработки**, не для продакшна: пароли простые, всё доступно лишь на твоём компьютере (`localhost`).
